Web Application Security

Rise of the Web App

Over the past few years we have seen tremendous growth in the rise of browser-based applications. Native applications are still popular however the trend is moving towards web apps that run directly from the browser.

Web applications are convenient by the fact that all data can be synced between devices and stored in the cloud. Gone are the days of configuring, updating and syncing the data on native apps. The explosive growth in web application development appears to be fueled by a rise in frameworks such as node.js, react, Django, Flask, and others. The barrier to entry to creating powerful browser-based applications is significantly lower.

The Dark Side

The rapid innovation has come with a cost by the fact that many web applications contain vulnerabilities that compromise the integrity of software leaving the company and the user vulnerable to loss of confidential data. The reason why they are vulnerable is because it is hard to configure a firewall to prevent malicious activities because the service must be available to its users.

Finding Vulnerabilities

A common tactic for exploring web application vulnerabilities is through scanning tools. Commercial vulnerability scanning tools offer automation of the scanning process. The Kali Linux Debian based operating system comes with numerous built in vulnerability testing tools.Learning the fundamentals of each tool will provide a foundation to successfully securing your web application.

The tools provided with Kali Linux are beyond the scope of this article. Kali does provide the tools necessary to test a web application. Knowing what to look for is important when starting a security test. According to theOpen Web Application Security Project (OWASP) below are the most common webapplication security vulnerabilities:

  1. Code injection: hackers find ways to insert malicious executable code into legitimate traffic sent to an endpoint
  2. Broken authentication and session management:compromising user identities in a variety of ways
  3. Cross-site scripting: similar to code injection,but involving scripts instead, drawn from inappropriate sources
  4. Insecure direct object references: obtaining file access when it’s not actually authorized
  5. Security misconfiguration: a failure of the admin, sometimes as simple as leaving passwords as defaults
  6. Sensitive data exposure: failure to shield detain proportion to its business value or customer sensitivity
  7. Missing function level access control: failure to verify functions are actually limited by access rights
  8. Cross-site request forgery: compromising an unexpected web application by leveraging validated authentication information
  9. Components with known vulnerabilities: a vulnerable element, such as a Java class, hasn’t been patched
  10. Unvalidated redirects and forwards: sending web users to unexpected sites that serve hacker interests

This is an extremely brief overview of SOME of the possible vulnerabilities.New risks are found daily. To get started a great book I recommend is The Web Application Hackers Handbook to gain more information on the subject. If you would like to practice testing applications on your own, experiment with Damn Vulnerable Web App and use penetration tools such as Burp, Grabber, Webscarab, Vega, etc.

The Future

Security measures will become more and more complex as time goes on. Everything from refrigerators to power plants is connected to the internet. Endpoints are diversifying, and everything is connected to one another. The popularity of cloud storage adds to the complexity as control over data is put into another companies hands. Security is only as strong as the weakest link. Security professionals will need to be more knowledgeable thanever to keep our data safe.
